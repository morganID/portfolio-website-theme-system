# Centralized Theme System

This portfolio website now uses a centralized theme system that allows you to manage colors, gradients, typography, spacing, and other design tokens from a single location.

## Structure

### `src/styles/theme.ts`
The main theme configuration file containing all design tokens organized by category:
- **Colors**: Primary brand colors, neutrals, semantic colors
- **Gradients**: Background gradients for sections and buttons
- **Typography**: Font sizes, weights, and line heights
- **Spacing**: Consistent spacing values
- **Border Radius**: Corner radius values
- **Shadows**: Box shadow definitions
- **Transitions**: Animation timing functions

### `src/styles/theme.css`
Auto-generated CSS variables based on the theme configuration. **Do not edit this file directly** - it should be regenerated when theme.ts changes.

### `src/styles/theme-utils.ts`
Utility functions for accessing theme values in components and helper functions for theme management.

## Usage

### In Components

Instead of hardcoded Tailwind classes or inline styles, use theme variables:

```tsx
// ❌ Old approach
<div className="bg-gradient-to-b from-gray-50 to-white">

// ✅ New approach
<div style={{ background: 'var(--gradient-hero)' }}>
```

### Available CSS Variables

#### Colors
- `--color-primary-50` through `--color-primary-900`
- `--color-purple-500`, `--color-purple-600`
- `--color-gray-50` through `--color-gray-900`
- `--color-white`, `--color-black`

#### Gradients
- `--gradient-primary`: Blue to purple gradient
- `--gradient-hero`: Hero section background
- `--gradient-button`: Button gradient

#### Typography
- `--font-size-xs` through `--font-size-7xl`
- `--font-weight-normal`, `--font-weight-medium`
- `--line-height-normal`, `--line-height-relaxed`

#### Spacing
- `--space-1` through `--space-24`

#### Other
- `--radius-sm` through `--radius-full`
- `--shadow-sm` through `--shadow-xl`
- `--transition-fast` through `--transition-slow`

## Modifying the Theme

1. **Edit `src/styles/theme.ts`** - Update the theme object with your desired values
2. **Update `src/styles/theme.css`** - Manually sync the CSS variables to match your theme.ts changes
3. **Test your changes** - Run `npm run dev` and verify the changes work as expected

### Example: Changing Primary Colors

```typescript
// In theme.ts
colors: {
  primary: {
    500: '#10b981', // Change from blue to green
    600: '#059669', // Darker green
    // ... other shades
  },
}
```

Then update the corresponding CSS variables in `theme.css`:

```css
--color-primary-500: #10b981;
--color-primary-600: #059669;
```

## Benefits

1. **Single Source of Truth**: All design tokens are defined in one place
2. **Consistency**: Ensures consistent styling across components
3. **Easy Maintenance**: Change theme values without hunting through multiple files
4. **Type Safety**: TypeScript definitions prevent invalid theme references
5. **Scalability**: Easy to add new theme properties as the project grows

## Migration Notes

Components that have been updated:
- ✅ `App.tsx` - Main background and footer colors
- ✅ `Hero.tsx` - Background gradient, name gradient, button gradient
- ✅ `Navigation.tsx` - Navigation bar styling

Other components can be migrated following the same pattern:
1. Replace hardcoded Tailwind classes with CSS custom properties
2. Use inline styles with `var(--theme-variable)` for complex values
3. Import React if needed for JSX
